AdminPanel API

A REST interface to a role-based access-control smart contract. Read roles, users and settings — and perform admin actions — over plain HTTP. No wallet or web3 library required.

What this is. Our smart contract decides who is an Owner, Moderator, or Regular User on-chain. This API puts a normal web layer in front of it, so any application can integrate by making standard HTTP requests instead of talking to the blockchain directly.

Permission tiers

RoleCan do
OWNEREverything: register/remove users, promote moderators, edit settings, moderate
MODERATORDeactivate/reactivate users, approve content, + all user reads
REGULAR_USERSelf-service actions (handled in the dApp, not this API)
NONERead public data only

Underlying contract: 0x18a5883dA6302e13021910235Cb74653b8804505 (verified on Etherscan).

Base URL & Versioning

All resource endpoints are served under /api/v1. Breaking changes ship under a new version prefix.

Base URL
https://your-host/api/v1

Authentication

Read endpoints

Open — no authentication. They only call view functions on the contract.

Write & admin endpoints

Require an API key sent in the x-api-key header. The server signs the transaction with an operator wallet.

HTTP header
x-api-key: YOUR_API_KEY
Note. Write access depends on the deployment. A public read-only instance returns 503 for write endpoints. Treat the API key like a password and always call over HTTPS.

Quickstart & Integration

Look up a wallet's role (open) and register a user (needs an API key). Pick your language:

example
# Read a wallet's role (no auth)
curl https://your-host/api/v1/roles/0xd7C1D2D96cc9cD8D9e90876f77c1DBeC8A33B857

# Register a user (needs API key; server signs the tx)
curl -X POST https://your-host/api/v1/users \
  -H "content-type: application/json" \
  -H "x-api-key: $API_KEY" \
  -d '{"address":"0x7E2C9...85d7d","username":"alice"}'
const BASE = "https://your-host";

// read
const { role } = await fetch(`${BASE}/api/v1/roles/${address}`).then(r => r.json());

// write
const res = await fetch(`${BASE}/api/v1/users`, {
  method: "POST",
  headers: { "content-type": "application/json", "x-api-key": API_KEY },
  body: JSON.stringify({ address, username: "alice" }),
});
const { txHash, explorerUrl } = await res.json();
import requests
BASE = "https://your-host"

role = requests.get(f"{BASE}/api/v1/roles/{address}").json()["role"]

res = requests.post(
    f"{BASE}/api/v1/users",
    headers={"x-api-key": API_KEY},
    json={"address": address, "username": "alice"},
)
print(res.json()["explorerUrl"])

Users

GET/api/v1/users keyList all users
GET/api/v1/users/count keyTotal registered users
GET/api/v1/users/{address}A user's public profile
GET/api/v1/users/{address}/activeActive status
POST/api/v1/users keyRegister a user (grants REGULAR_USER)

POST/api/v1/users/{address}/deactivate keyDeactivate a user
POST/api/v1/users/{address}/reactivate keyReactivate a user

Moderators · Settings · Content

Errors

Every error is JSON with a machine-readable code and a human message:

error shape
{ "error": { "code": "bad_request", "message": "…", "details": { } } }
StatuscodeMeaning
400bad_requestInvalid input (bad address, missing field)
401unauthorizedMissing or invalid API key
404not_foundUnknown route or unregistered user
409conflictOn-chain revert (e.g. “User does not exist”)
429rate_limitedToo many requests (120/min per IP)
502upstream_errorBlockchain RPC unreachable
503service_unavailableWrites disabled, or operator out of gas
